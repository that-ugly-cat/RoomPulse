RoomPulse — user guide

Live polling for talks and lectures. Your audience answers from their phones; results appear as you speak. This guide walks a presenter through building a deck, running a live session, and using every question type.

What it is

RoomPulse is a self-hosted, Mentimeter-style tool built for keynotes and teaching — with a focus on argumentative and ethical questions, not just trivia. Three ideas shape it:

  • The audience follows you. No push: their screen shows whatever slide you've made active. You get manual reveal and stage control for free.
  • Zero friction. No app, no login for the audience — a 5-digit code or a QR code, and they're in.
  • Quantitative vs. moderated. Numeric types draw a chart; free-text types pass through a presenter-side moderation queue.

Getting started

  1. Open the app and register (or sign in) on the login page. Your account owns your decks — you only ever see your own.
  2. You land in the editor (your dashboard). Create a deck with the + next to the title field on the left.
  3. Add slides, then open the presenter view to project and drive the session.
  4. Share the join code or QR with the room. That's it.

The UI is available in Italian, English and German — switch with the language selector on any page.

Building a deck

A deck is a reusable template of slides. In the editor:

  • Add a slide — pick a type, write the question, fill in the type-specific config (options, labels, min/max…), then Add.
  • Reorder — drag the ≡ handle.
  • Inspect — click a slide to expand its config; rename the deck with the ✎.
  • Export / import a deck as JSON to move it between accounts or back it up.

The same deck can be run many times. Each run keeps its own responses, so you can reuse a deck across classes or sessions without mixing the data.

The audience follows the presenter via light polling (about once a second) — no websockets, nothing to install.

Running a session

  1. Open the presenter view for a deck (from the editor, Presenter ↗).
  2. Click a slide on the left to activate it. The first activation starts a run automatically; use New run to start a fresh one (e.g. a new class).
  3. The audience is now answering. Watch the count and results update live.
  4. Control the slide state:
    • Show results / Hide — reveal the chart to the audience (the classic dramatic reveal).
    • Reopen voting — let people change or add answers.

Q&A and Donut use open / closed instead of reveal: the feed / leaderboard is always visible, and you just open or close submissions.

Pre / post

Ask the same question before and after your argument and show how the room moved. When adding a slide of type scale, choice or quadrant, link it as the POST of an earlier twin. The results view overlays the two and draws the shift: the change in mean for scales, per-option deltas for choice, and an arrow between the centroids for the quadrant.

This is the strong move for ethical topics — you measure how much the argument actually shifted positions.

Quiz mode

Turn a choice question into a quiz: mark the correct option(s). Participants find out whether they were right only after they answer, their answer locks (no changing it), and the correct answer is never leaked before the reveal. Quiz mode and participant-added options are mutually exclusive.

Random groups

The randomizer splits the room into named groups. Assignment is balanced (it fills the smallest group next) and stable per participant — refreshing keeps you in the same group. Handy for break-out activities and debates.

Donut energizer

Not a question but a break that also produces data: a one-button endless-runner (tap anywhere or space to jump) that runs on the audience's phones. Dodge the ruins, grab donuts (+100 each), and everyone's best scores build a live leaderboard. Each player keeps their best score; retries are free. Useful when the talk is about attention or human factors — the real distribution of scores becomes a talking point. Open / closed works like Q&A: while open people play and see the top few; when you close it the full leaderboard shows.

AI argument clustering

For Claim + justification and Open text, an LLM can group free responses into descriptive themes — for claim+justification along two axes (the criterion and the kind of appeal, with a criterion × appeal matrix). It's descriptive, never evaluative.

  1. Open the editor, click ⚙, and paste your own Anthropic API key (stored per user). Get one at console.anthropic.com.
  2. During the session, click Cluster on that slide. Re-run any time to refresh as more answers come in.

Moderation

Free-text types (open text, Q&A, claim+justification) show a moderation queue next to the results on the presenter view. Answers are visible by default; hide a troll with one click, show it again if you change your mind. The audience only ever sees visible answers.

Export & import

  • Deck (JSON) — Export deck / Import deck moves the template (slides + config, pre/post links preserved) between accounts.
  • Data (Excel) — Export data downloads an .xlsx workbook for the deck: an Overview sheet listing every run, then one sheet per run with the raw responses (one row each, resolved labels, claim/justification and cluster tags in their own columns) — the full history in one file, ready for a paper or a spreadsheet.

Tips & troubleshooting

  • One fixed code per deck. The join code lives on the deck, not the run — the audience always uses the same code / QR, and the backend routes it to the current run.
  • Nothing on screen? The audience only sees a question once you've activated one. Until then they see a waiting screen.
  • Quiz answer won't change. That's by design — quiz answers lock after the first submit.
  • Back up your data by exporting the Excel workbook before you delete a deck; deletion is permanent and takes runs, responses and votes with it.
  • Privacy. The audience is anonymous (a lightweight browser token, no login). Treat the join code as the only gate.
